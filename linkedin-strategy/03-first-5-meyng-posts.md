## POST 4 — Behind the Tech (Engineering)

**When:** Day 10-11
**Visual:** Code snippet screenshot or architecture diagram

```
Building AI for underserved communities comes with unique technical challenges. Here's what powers SangoAI under the hood:

The stack:
- Frontend: React + TypeScript + Vite (deployed on Vercel)
- Backend: AWS Lambda (Python) — fully serverless
- AI: Claude 3.5 Sonnet via AWS Bedrock for translation and chat
- Auth: AWS Cognito with Google OAuth
- Database: DynamoDB for sessions and usage tracking
- Payments: Stripe + MTN Mobile Money (for African users)

The hardest part? Training AI to understand a language with limited digital corpus. Sango has almost no presence in existing AI training data. We had to build custom approaches to make translation accurate and culturally relevant.

And we designed everything to work as a PWA — because many of our users access the web on low-bandwidth connections with basic devices.

Technology choices matter. We chose every tool with our users' constraints in mind.

#SangoAI #Serverless #AWS #React #TypeScript #AIEngineering #TechForGood #MEYNG
```

## Posting Tips

1. **Post between 8-10 AM Paris time** (Tuesday-Thursday get highest engagement)
2. **Add 3-5 hashtags** per post (not more — LinkedIn penalizes hashtag stuffing)
3. **First line is everything** — LinkedIn shows only 2 lines before "...see more". Make them compelling.
4. **Engage with comments** within the first hour — this signals LinkedIn to boost the post
5. **No external links in the post body** if possible — LinkedIn deprioritizes posts with links. Put the URL in the first comment instead, or use the "Featured" section.
6. **Use line breaks generously** — walls of text get scrolled past
